# Backend Development – Day 18 Notes

## What is MVC Architecture?

MVC stands for Model–View–Controller. It is a design pattern used to organize code in applications.

MVC Structure:

Client Request  
→ Routes  
→ Controller  
→ Model  
→ Database  
→ Response

## Components of MVC

Model  
Responsible for managing data and interacting with the database.

Controller  
Contains the business logic and handles requests from routes.

Routes  
Define API endpoints and connect them to the appropriate controller functions.

## Advantages of MVC

- Better code organization
- Separation of concerns
- Easier debugging and maintenance
- Scalable for large applications
