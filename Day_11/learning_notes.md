# Backend Development – Day 11 Notes

## What is a REST API?

REST (Representational State Transfer) is an architectural style used for designing networked applications.

REST APIs allow communication between client and server using HTTP requests.

## REST API Components

### 1. Endpoint (URL)

Example:

/api/users

### 2. HTTP Methods

GET – Retrieve data  
POST – Create new data  
PUT – Update existing data  
DELETE – Remove data  

### 3. Headers

Contain metadata such as content type and authorization.

### 4. Body

Contains data sent to the server (mostly used with POST and PUT).

## HTTP Status Codes

200 – Success  
201 – Created  
400 – Bad Request  
404 – Not Found  
500 – Server Error  

## API Testing Tools

Postman is commonly used to test APIs by sending HTTP requests and viewing responses.
