# Backend Development – Day 23 Notes

## What is API Documentation?

API documentation explains how to use an API, including endpoints, request methods, parameters, and responses.

## Key Components

- Endpoint (URL)
- HTTP Method (GET, POST, etc.)
- Request Body
- Response Format
- Status Codes

## Importance

- Helps frontend developers integrate APIs
- Improves collaboration
- Makes APIs easy to understand and use

## Example

GET /tasks → Fetch tasks  
POST /tasks → Add new task  

## Learning Outcome

Today I learned how to create clear and structured API documentation to improve usability and communication between frontend and backend developers.
