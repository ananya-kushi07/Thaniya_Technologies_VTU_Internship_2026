# Backend Development – Day 24 Notes

## What is Deployment?

Deployment is the process of making an application available for users over the internet.

## Steps in Deployment

1. Prepare project code
2. Configure environment variables
3. Choose hosting platform (Render, Vercel, etc.)
4. Run application on server

## Local vs Remote Server

Local Server  
Runs on your system (localhost)

Remote Server  
Runs on cloud and accessible via internet

## Environment Variables

Used to store:
- Port number
- API keys
- Database URLs

Example:
process.env.PORT

## Benefits

- Secure data handling
- Easy configuration changes
- Production-ready applications

## Learning Outcome

Today I learned how backend applications are prepared for deployment and how environment variables help in secure and flexible configuration.
