# Backend Development – Day 16 Notes

## What is MongoDB?

MongoDB is a NoSQL database that stores data in a flexible JSON-like format called documents.

Instead of tables and rows, MongoDB uses:

Database
→ Collections
→ Documents

## Key Concepts

Document  
A single record stored in JSON-like format.

Collection  
A group of related documents.

Schema  
Defines the structure of documents in a collection.

## Basic Operations

Insert Data  
Adding new documents into a collection.

Find Data  
Retrieving documents from the database.

Update Data  
Modifying existing documents.

Delete Data  
Removing documents from a collection.

## Learning Outcome

Today I learned how MongoDB stores and manages data using collections and documents. I also understood how backend applications interact with MongoDB to store and retrieve data efficiently.
