# Day 10 – Backend Project Structure using MVC Architecture

On Day 10, I learned about organizing backend applications using the MVC (Model-View-Controller) architecture. Instead of writing all code in a single file, the backend can be structured into separate layers to improve readability, scalability, and maintainability.

I explored how models represent the database structure, controllers handle business logic, and routes manage API endpoints. This separation helps developers build large applications in a clean and structured way.

Through this learning, I understood how professional backend systems are organized and how modular coding practices improve project development.
