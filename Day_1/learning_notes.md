# Backend Development – Day 1 Notes

## What is Backend?

Backend refers to the server-side of a web application. It handles:

- Business logic
- Database operations
- Authentication
- API responses

## Why Node.js?

- Allows JavaScript to run on the server
- Fast and scalable
- Uses event-driven, non-blocking I/O

## What is Express.js?

Express.js is a lightweight Node.js framework used to:

- Create servers easily
- Handle routing
- Manage middleware
- Build REST APIs

## HTTP Methods Learned

- **GET** – Fetch data
- **POST** – Send data
- **PUT** – Update data
- **DELETE** – Remove data

## Folder Structure Basics

- **server.js** – Entry point
- **routes** – Handles routing
- **controllers** – Business logic
- **models** – Database schemas (to be learned later)

## Key Learnings

- How client-server communication works
- How to start a Node.js server
- Importance of modular backend structure
